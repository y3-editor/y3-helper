# 04 - 启动流程详解

## 1. 扩展激活入口

VSCode 加载扩展后调用 `src/extension.ts` 中导出的 `activate()` 函数。

```typescript
// src/extension.ts
export async function activate(context: vscode.ExtensionContext) {
    // ... 初始化逻辑
}
```

---

## 2. 完整启动流程

```
VSCode 检测到激活条件（工作区含 y3/更新日志.md 或 y3/CHANGELOG.md）
    │
    ▼
activate(context) 被调用
    │
    ├─① 设置模块别名
    │     moduleAlias.addAliases({ 'y3-helper': __dirname + '/y3-helper' })
    │     （使代码中 import * as y3 from 'y3-helper' 能正确找到模块）
    │
    ├─② 设置 y3 上下文
    │     y3.setContext(context)
    │
    ├─③ 检测并设置语言
    │     读取配置 Y3-Helper.Language
    │     若为 default，检查 vscode.env.language 或系统 locale
    │     若非中文，加载 l10n/bundle.l10n.json 翻译文件
    │
    ├─④ 创建 Helper 实例
    │     let helper = new Helper(context)
    │
    └─⑤ 调用 helper.start()
              │
              ├─ 注册命令（同步，立即执行）
              │     registerCommandOfInitProject()    → y3-helper.initProject
              │     registerCommandOfMakeLuaDoc()     → y3-helper.makeLuaDoc
              │     registerCommandOfLaunchGame()     → y3-helper.launchGame
              │     registerCommandOfAttach()         → y3-helper.attach
              │     registerCommandOfLaunchEditor()   → y3-helper.launchEditor
              │     reloadEnvWhenConfigChange()       → 监听配置变化
              │     registerCommandOfNetworkServer()  → y3-helper.networkServer
              │     registerCommonCommands()          → y3-helper.selectAnotherMap 等
              │
              └─ 延迟 100ms 后执行子模块初始化（异步）
                        │
                        ├─ checkNewProject()     检查是否刚初始化了新项目
                        ├─ mainMenu.init()       初始化左侧主菜单树形视图
                        ├─ metaBuilder.init()    初始化 Meta 文件生成器
                        ├─ debug.init(context)   初始化 Lua 调试器
                        ├─ console.init()        启动游戏通信 TCP 服务器
                        ├─ editorTable.init()    初始化物编数据视图
                        ├─ plugin.init()         初始化插件系统
                        ├─ globalScript.init()   初始化全局脚本监听
                        ├─ luaLanguage.init()    初始化 Lua 语言服务
                        ├─ ecaCompiler.init()    注册 ECA 编译命令
                        └─ y3.version.init()     初始化版本检查
```

> **为什么延迟 100ms？**  
> 这是为了避免在 VSCode 完全初始化之前就开始耗时的文件系统操作，给宿主环境一点时间稳定下来。

## 3. 环境（Env）的惰性初始化

`env` 对象是单例（`src/env.ts`），它不在 `activate` 时立即搜索编辑器和地图，而是**按需惰性加载**。

```
各模块调用 env.mapReady() / env.editorReady()
    │
    ▼
若 projectUri / editorUri 已存在 → 直接返回
    │
    ▼
若不存在 → 开始搜索
    │
    ├─ searchProject(search=true, askUser=false)
    │     遍历所有 workspace folders
    │     向上最多搜索 20 层，寻找含 header.project 的目录
    │
    └─ 找到后执行 project.start()
              │
              ├─ 读取 header.project → 获取 entryMapId
              ├─ 遍历 maps/ 目录，创建所有 Map 对象
              ├─ 每个 Map.start() 并行执行：
              │     读取 header.map → 获取 map.id
              │     读取 desc.json → 获取 map.description
              │     创建 .log 目录
              └─ 找到 entryMap（id 匹配的地图）
                        │
                        ▼
              env.updateCurrentMap(entryMap)
                  更新所有路径快捷属性：
                  mapUri / scriptUri / y3Uri /
                  helperUri / metaUri / pluginUri /
                  excelUri / ruleUri
                        │
                        ▼
              env.onDidChange 事件触发
              （所有监听环境变化的模块开始工作）
```

---

## 4. env.onDidChange 事件的订阅者

当环境发生变化（地图切换、配置更新等），以下模块会收到通知并更新：

| 模块 | 响应行为 |
|------|---------|
| `mainMenu` | 刷新左侧菜单树 |
| `metaBuilder` | 重新生成所有 meta 文件 |
| `console` | 更新 helper_port.lua 到新地图 |
| `debug` | 更新 debugger_path.lua 到新地图 |
| `globalScript` | 检查全局脚本是否启用，更新 .luarc.json |

---

## 5. 子模块初始化详情

### mainMenu.init()

```
创建 MainMenu 实例
    ├─ makeMainNode() 构建树形菜单结构（功能/地图管理/插件列表/枚举/环境...）
    ├─ 创建 TreeProvider（实现 vscode.TreeDataProvider）
    ├─ vscode.window.createTreeView('y3-helper.mainMenu', ...)
    ├─ 监听 view 可见性变化 → 刷新
    ├─ 监听 checkbox 状态变化 → 回调节点处理函数
    └─ 监听 env.onDidChange → 刷新菜单
```

### metaBuilder.init()

```
创建 InitBuilder 实例（路径：meta/init.lua）
    ├─ 注册各子 builder：
    │     UnitAttrs    → meta/unitAttrs.lua
    │     PlayerAttrs  → meta/playerAttrs.lua
    │     CustomEvents → meta/customEvents.lua
    │     UI           → meta/ui.lua
    │     UIAnim       → meta/uiAnim.lua
    │     JumpWord     → meta/jumpword.lua
    │     Font         → meta/font.lua
    ├─ 订阅 env.onDidChange → 100ms 后 updateAll()
    │     每个 builder 读取对应的 customDefine 数据
    │     生成 Lua meta 文件（include 语句集合）
    └─ 创建 TS 实例（plugin/map-declare.d.ts）
         监听 customDefine 数据变化，生成 TypeScript 类型声明
```

### debug.init(context)

```
加载 3rd/debugger/js/extension.js（Lua 调试器）
    ├─ 创建伪造的 debuggerContext（修改 extensionUri 指向 3rd/debugger）
    ├─ 调用调试器的 activate(debuggerContext)
    ├─ 更新各地图的 debugger_path.lua（游戏启动时读取调试器路径）
    ├─ 创建状态栏按钮（✨启动 / 💡附加）
    └─ 启动 startWaitDebuggerHelper()（每秒检测是否需要重新附加调试器）
```

### console.init()

```
注册所有 RPC 方法（print / createTreeView / command / showInputBox 等）
    ├─ 读取配置 Y3-Helper.ServerPort
    │     若为 0 则随机选择 10000-65535 之间的端口
    ├─ 创建 ConsoleServer（TCP 服务器）监听该端口
    ├─ 订阅 env.onDidChange → 更新 helper_port.lua
    └─ 立即写入当前 helper_port.lua 到所有地图
         （游戏启动时读取此文件，连接到正确端口）
```
