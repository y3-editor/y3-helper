# 02 - 编译与打包

## 1. 编译方式概述

本项目有两套"编译"流程，用途不同：

| 方式 | 命令 | 工具 | 输出目录 | 用途 |
|------|------|------|---------|------|
| TypeScript 编译 | `tsc -p ./` | tsc | `out/` | 测试运行、调试 |
| Webpack 打包 | `npm run compile` | Webpack 5 | `dist/` | 实际发布，VSCode 加载 |

---

## 2. TypeScript 配置（tsconfig.json）

```json
{
  "compilerOptions": {
    "module": "Node16",         // 模块格式：Node16（支持 ESM/CJS）
    "target": "ES2022",         // 编译目标：ES2022
    "outDir": "out",            // 编译输出到 out/ 目录
    "lib": ["ES2022"],          // 使用 ES2022 标准库
    "strict": true,             // 严格模式
    "skipLibCheck": true,       // 跳过 node_modules 类型检查（避免第三方库类型冲突）
    "sourceMap": true,          // 生成 source map
    "declaration": true,        // 生成 .d.ts 类型声明
    "declarationDir": "./tmp/types",
    "experimentalDecorators": true  // 启用装饰器（@queue, @throttle 等）
  }
}
```

> **注意**：`skipLibCheck: true` 是为了解决 `lru-cache` 等第三方库类型定义与新版 TypeScript 不兼容的问题。这不影响对 `src/` 目录内代码的类型检查。

## 3. Webpack 5 打包配置（webpack.config.js）

Webpack 将 `src/extension.ts` 作为入口，把所有依赖打包成单一文件 `dist/extension.js`。

关键配置说明：

```javascript
module.exports = {
  target: 'node',               // 目标环境是 Node.js（非浏览器）
  entry: './src/extension.ts',  // 入口文件
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'extension.js',
    library: { type: "commonjs2" },  // 输出为 CJS 模块（VSCode 要求）
    devtoolModuleFilenameTemplate: "../[resource-path]",
  },
  externals: {
    vscode: 'commonjs vscode'   // vscode 模块由 VSCode 宿主提供，不打包进去
  },
  resolve: {
    extensions: ['.ts', '.js'],
    plugins: [new TsconfigPathsPlugin()]  // 支持 tsconfig.json 中的 paths 别名
  },
  module: {
    rules: [{
      test: /\.ts$/,
      use: 'ts-loader'           // 用 ts-loader 处理 TypeScript 文件
    }]
  },
}
```

**为什么用 Webpack 而不是直接用 tsc？**  
- VSCode 扩展加载的是 `package.json` 中 `"main"` 指定的文件（`dist/extension.js`）
- Webpack 将所有 `node_modules` 依赖也打包进去，使扩展无需用户安装依赖就能运行
- 对 `vscode` 模块做 `externals` 处理，因为它由 VSCode 宿主提供

## 4. 常用编译命令

```bash
# 开发时：以非优化模式打包（包含 source map，方便调试）
npm run compile
# 等价于：webpack --mode none

# 监听模式：文件变化后自动重新打包
npm run watch
# 等价于：webpack --mode none --watch

# 发布前：以生产模式打包（压缩代码）
# 由 vsce package 自动调用，对应 package.json 中的 vscode:prepublish
npm run vscode:prepublish
# 等价于：webpack --mode production

# 代码检查
npm run lint
# 等价于：eslint src --ext ts
```

## 5. 打包为 .vsix 安装包（内网版）

内网版不走 GitHub Actions，需要手动打包 `.vsix` 文件供他人安装。

### 步骤一：安装打包工具

```bash
npm install -g @vscode/vsce
```

### 步骤二：打包

```bash
# 在项目根目录执行
vsce package
```

> `vsce package` 在打包前会自动调用 `npm run vscode:prepublish`（即 `webpack --mode production`），因此无需手动编译。

执行后会在项目根目录生成 `y3-helper-x.x.x.vsix` 文件（版本号取自 `package.json` 的 `version` 字段）。

### 步骤三：安装 .vsix

方法一：VSCode 命令面板
1. 按 `Ctrl+Shift+P`，搜索 `Install from VSIX`
2. 选择生成的 `.vsix` 文件

方法二：命令行
```bash
code --install-extension y3-helper-x.x.x.vsix
```

## 6. 依赖管理

### 生产依赖（dependencies）

这些依赖会被 Webpack 打包进 `dist/extension.js`，最终用户不需要手动安装：

| 包 | 用途 |
|----|------|
| `exceljs` | 读取 Excel 文件（导表功能） |
| `fast-xml-parser` | 解析 XML |
| `fs-extra` | 扩展的文件系统操作 |
| `iconv-lite` | 字符编码转换 |
| `jsonc-parser` | 解析带注释的 JSON（JSONC） |
| `jszip` | ZIP 文件处理 |
| `module-alias` | 模块路径别名（开发时 `y3-helper` 别名） |
| `os-locale` | 获取系统语言（用于判断显示中文还是英文） |
| `uuid` | 生成唯一 ID |
| `winreg` | 读取 Windows 注册表（查找编辑器路径） |
| `@vscode/l10n` | 国际化翻译运行时 |

### 开发依赖（devDependencies）

仅在开发时使用，不会被打包：

| 包 | 用途 |
|----|------|
| `webpack` / `webpack-cli` | 打包工具 |
| `ts-loader` | Webpack 的 TypeScript 处理器 |
| `typescript` | TypeScript 编译器 |
| `eslint` / `@typescript-eslint/*` | 代码规范检查 |
| `@types/*` | 各库的 TypeScript 类型声明 |
| `mocha` | 测试框架 |
| `@vscode/vsce` | VSCode 扩展打包/发布工具 |

### 版本覆盖（overrides）

为修复间接依赖中的安全漏洞，使用 `overrides` 字段强制指定版本：

```json
"overrides": {
    "serialize-javascript": ">=7.0.3",
    "diff": ">=8.0.3"
}
```
