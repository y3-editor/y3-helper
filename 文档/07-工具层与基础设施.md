# 07 - 工具层与基础设施

## 1. tools 模块（src/tools/）

提供整个项目共用的基础工具函数。

### fs.ts — 文件系统工具

对 VSCode 的 `workspace.fs` API 做了简化封装：

```typescript
// 读取文件（返回包含 string 和 buffer 属性的 File 对象）
readFile(uri, relativePath?) → File | undefined

// 写入文件（text 或 buffer）
writeFile(uri, relativePath?, data) → boolean

// 删除文件/目录
removeFile(uri, relativePath?, options?) → boolean

// 检查文件是否存在
isExists(uri, relativePath?) → boolean

// 获取文件/目录状态
stat(uri, relativePath?) → FileStat | undefined

// 列出目录内容
dir(uri, relativePath?) → [name, FileType][]

// 递归扫描目录
scan(uri, relativePath?) → [relativePath, FileType][]

// 创建目录
makeDir(uri, relativePath?) → boolean

// 复制文件/目录
copy(from, to, options?) → boolean
```

**File 类**：readFile 返回的对象，同时提供 `string`（文本）和 `buffer`（二进制）视图。

### log.ts — 日志工具

将日志同时输出到 VSCode 输出面板和文件：

```typescript
log.info('信息')    // 普通日志
log.warn('警告')    // 警告日志
log.error('错误')   // 错误日志
```

日志文件路径：`{scriptUri}/.log/helper.log`

### json.ts — JSON 工具

提供带位置信息的 JSONC（带注释 JSON）操作，支持：
- 保留原有格式（缩进、注释）进行修改
- 通过路径访问嵌套字段（如 `'runtime.path'`）

```typescript
class Json {
    get(path: string): any           // 读取字段
    set(path: string, value: any)    // 设置字段（保留格式）
    get text(): string               // 获取修改后的文本
}
```

### y3json.ts — Y3 自定义 JSON 格式

Y3 的物编 JSON 文件使用特殊格式（支持 bigint、tuple 等类型），此模块处理这些扩展类型的序列化和反序列化。

### version.ts / versionCache.ts — 版本管理

管理 y3-lualib 版本信息，支持版本比较和缓存。

### download.ts — 下载工具

HTTP/HTTPS 文件下载工具。

### lua.ts — Lua 工具

处理 Lua 代码生成的工具函数。

### tracy.ts — Tracy 性能分析

启动 Tracy 性能分析器。

## 2. utility 模块（src/utility/）

通用编程工具，与业务逻辑无关。

### decorators.ts — TypeScript 装饰器

项目中大量使用这两个装饰器：

**@queue()** — 串行化异步函数，防止并发执行

```typescript
class Env {
    @queue()
    async updateMap(search: boolean, askUser: boolean) {
        // 即使被多次调用，也会排队串行执行
        // 避免并发修改 project 对象
    }
}
```

适用场景：需要保证执行顺序的操作，如文件读写、环境初始化。

**@throttle(ms)** — 节流，在指定时间内合并多次调用

```typescript
class Env {
    @throttle(100)
    private fireOnDidReload() {
        // 100ms 内多次调用，只触发一次
        this.envChangeEmitter.fire();
    }
}
```

适用场景：频繁触发的事件，如文件变化通知、UI 刷新。

### spinLock.ts — 自旋锁

异步自旋锁，用于需要互斥访问的场景：

```typescript
const lock = new SpinLock();
await lock.acquire();
try {
    // 临界区
} finally {
    lock.release();
}
```

### hashSet.ts — 哈希集合

基于对象哈希的 Set 实现，支持自定义哈希函数。

### utilityFunctions.ts — 通用工具函数

包含 `randomInt`（随机整数）等常用函数。

## 3. 国际化（l10n）

项目支持中文（默认）和英文两种语言。

### 使用方式

```typescript
import * as l10n from '@vscode/l10n';

// 在代码中使用 l10n.t() 包裹需要翻译的字符串
vscode.window.showErrorMessage(l10n.t('未找到Y3地图路径，请先用编辑器创建地图！'));
```

### 翻译文件

- 中文是默认语言，不需要翻译文件
- 英文翻译：`l10n/bundle.l10n.json`

```json
{
    "未找到Y3地图路径，请先用编辑器创建地图！": "Y3 map path not found, please create a map in the editor first!"
}
```

### 语言检测逻辑

```typescript
// activate() 中
let language = vscode.workspace.getConfiguration('Y3-Helper').get('Language');
if (language === 'default') {
    // VSCode 语言 或 系统语言 任意一个是中文，则使用中文
    if (vscode.env.language === 'zh-cn' || await osLocale.osLocale() === 'zh-CN') {
        language = 'zh-cn';
    } else {
        language = 'en';
    }
}
```

## 4. 模块别名系统（y3-helper 别名）

项目中大量使用 `import * as y3 from 'y3-helper'`，这不是一个 npm 包，而是项目内部的模块。

### 原理

1. **TypeScript 层**（`tsconfig.json`）：
   ```json
   "paths": {
       "y3-helper": ["./src/y3-helper"],
       "map-declare": ["./src/map-declare"]
   }
   ```

2. **Webpack 层**（`tsconfig-paths-webpack-plugin`）：
   自动读取 tsconfig 的 paths 配置，在打包时也能正确解析别名。

3. **运行时层**（`extension.ts` 最顶部）：
   ```typescript
   import moduleAlias from 'module-alias';
   moduleAlias.addAliases({
       'y3-helper': __dirname + '/y3-helper'
   });
   ```
   这是给打包后的 `dist/extension.js` 用的，因为 Webpack 打包后路径会变化。

### y3-helper 模块提供的 API

`src/y3-helper.ts` 是一个聚合导出文件，将所有工具集中暴露：

- `y3.fs` — 文件系统工具
- `y3.log` — 日志工具
- `y3.json` — JSON 工具
- `y3.env` — 环境对象（等同于 env 单例）
- `y3.uri(...)` — URI 路径拼接工具
- `y3.sleep(ms)` — 异步等待
- `y3.plugin` — 插件相关 API
- `y3.editorTable` — 物编数据访问 API
- `y3.excel` — Excel 工具
- `y3.version` — 版本管理
- `y3.tracy` — Tracy 工具
- `y3.helper` — VSCode 扩展上下文

## 5. 命令注册约定

所有 VSCode 命令都在 `package.json` 的 `contributes.commands` 中声明，并在代码中注册：

```typescript
// 命令 ID 格式：y3-helper.{功能名称}
vscode.commands.registerCommand('y3-helper.initProject', async () => {
    // 处理函数
});
```

命令可以通过以下方式触发：
1. 命令面板（`Ctrl+Shift+P`）
2. 菜单项（在 TreeNode 的 `command` 属性中配置）
3. 右键菜单（在 `package.json` 的 `contributes.menus` 中配置）
4. 键盘快捷键（在 `package.json` 的 `contributes.keybindings` 中配置）

## 6. 错误处理约定

项目采用"不抛异常，返回 undefined/false"的风格处理可预期错误：

```typescript
// 典型模式
async function readFile(uri: vscode.Uri) {
    try {
        let data = await vscode.workspace.fs.readFile(uri);
        return new File().write(data);
    } catch {
        return undefined;  // 文件不存在时返回 undefined，不抛异常
    }
}
```

用户可见的错误通过 VSCode 的消息 API 展示：
```typescript
vscode.window.showErrorMessage('错误信息');    // 错误（红色）
vscode.window.showWarningMessage('警告信息');  // 警告（黄色）
vscode.window.showInformationMessage('信息');  // 信息（蓝色）
```
