# 05 - 核心模块详解

## 1. env 模块（src/env.ts）

`env` 是全局单例，储存了所有运行时环境信息。几乎所有模块都依赖它。

### 关键数据结构

```typescript
class Env {
    language: 'zh-cn' | 'en'       // 当前语言
    editorUri?: vscode.Uri          // Y3 Editor.exe 路径
    editorExeUri?: vscode.Uri       // Game_x64h.exe 路径
    editorVersion: '1.0'|'2.0'|'unknown'  // 编辑器版本
    projectUri?: vscode.Uri         // 工程根目录（含 header.project 的目录）
    mapUri?: vscode.Uri             // 当前地图目录
    scriptUri?: vscode.Uri          // 当前地图的 script/ 目录
    y3Uri?: vscode.Uri              // script/y3/ 目录（Lua 库）
    helperUri?: vscode.Uri          // script/y3-helper/ 目录（助手数据）
    metaUri?: vscode.Uri            // script/y3-helper/meta/
    pluginUri?: vscode.Uri          // script/y3-helper/plugin/
    excelUri?: vscode.Uri           // script/y3-helper/excel/（Excel 文件存放处）
    ruleUri?: vscode.Uri            // script/y3-helper/excel_rule/（导表规则）
    project?: Project               // 当前工程对象
    currentMap?: Map                // 当前选中的地图
}
```

### 编辑器路径搜索优先级

1. VSCode 配置 `Y3-Helper.EditorPath`
2. Windows 注册表 `HKLM\SOFTWARE\Classes\y3editor`
3. 以上都没有 → 弹出文件选择对话框（仅 askUser=true 时）

### Project 与 Map 层级关系

```
Project（工程）         ← header.project 标识
├── maps/
│   ├── Map A           ← header.map 标识（有 id）
│   ├── Map B
│   └── Map C（入口地图 entryMap）
└── global_script/      ← 全局脚本目录
```

### 多地图时的"主地图触发器"特性

当工程设置中启用了 `use_main_level_trigger_and_object` 时，所有地图共享入口地图的物编数据和触发器：

```typescript
get editorTable() {
    if (env.project?.setting?.use_main_level_trigger_and_object) {
        return env.project.entryMap._editorTable;  // 使用主地图的物编
    } else {
        return this._editorTable;                   // 使用自己的物编
    }
}
```

## 2. mainMenu 模块（src/mainMenu/）

左侧活动栏的树形视图面板，包含所有用户可操作的功能入口。

### 树形结构

```
主菜单
├── 功能 (features.ts)
│     ├── 初始化项目
│     ├── 启动游戏（含调试模式设置）
│     ├── 生成 Lua 文档
│     └── ECA 编译
├── 地图管理 (mapManager.ts)
│     ├── 选择启动地图（当前/入口/指定地图）
│     └── 多开模式设置
├── 插件列表 (plugin.ts)
│     └── 显示 plugin/ 目录下所有 .js 文件
├── 界面 (ui.ts)
│     └── 画板 / 场景UI / 元件 列表
├── 枚举
│     ├── 单位属性 (unitAttrs.ts)
│     ├── 玩家属性 (playerAttrs.ts)
│     ├── 自定义事件 (events.ts)
│     ├── 时间轴动画 (uiAnim.ts)
│     ├── 跳字 (jumpword.ts)
│     └── 字体 (font.ts)
├── 环境 (environments.ts)
│     ├── 全局脚本开关
│     └── 其他环境设置
└── 重新选择Y3地图路径
```

### TreeNode 基类

每个菜单项都继承自 `TreeNode`，支持：
- `show()` — 异步判断是否显示该节点
- `childs` — 子节点列表
- `command` — 点击时执行的 VSCode 命令
- `onDidChangeCheckboxState` — Checkbox 状态变化回调
- `iconPath` — 图标

## 3. editorTable 模块（src/editorTable/）

管理 Y3 物编数据（`editor_table/` 目录下的 JSON 文件）。

### 文件结构

```
{mapDir}/editor_table/
├── unit/                  # 单位
│   ├── 100001.json
│   └── 100002.json
├── ability/               # 技能
├── item/                  # 道具
├── modifier/              # 修饰器
├── projectile/            # 投射物
├── destructible/          # 可破坏物
├── tech/                  # 科技
├── sound/                 # 音效
└── decoration/            # 装饰物
```

### 关键类

**EditorManager** — 管理一个地图的所有物编表
```typescript
class EditorManager {
    tables: Map<Table.NameCN, EditorTable>  // 各类型物编表的集合
    
    getTable(name: Table.NameCN): EditorTable
    getAllObjects(): EditorObject[]
}
```

**EditorTable** — 单个物编表（如 unit 表）
```typescript
class EditorTable {
    objects: Map<number, EditorObject>  // key → 物编对象
    
    create(key?: number): EditorObject  // 新建物编对象
    delete(key: number)                  // 删除
    getByName(name: string): EditorObject[]
}
```

**EditorObject** — 单个物编数据条目
```typescript
class EditorObject {
    tableName: Table.NameCN  // 所属表类型
    key: number              // 唯一 key
    
    get(field: string): any              // 读取字段值
    set(field: string, value: any)       // 设置字段值
    getName(): string                    // 获取名称
    save()                               // 保存到 JSON 文件
}
```

### 字段元数据（helper_meta/editor/）

每类物编表都有对应的 `.json` 文件描述字段信息：
```json
// helper_meta/editor/unit.json
{
    "field_name": {
        "key": "actual_key_in_json",
        "type": "string|number|boolean",
        "desc": "字段描述",
        "tips": "字段提示"
    }
}
```

### 物编树形视图（treeView.ts）

注册为 `y3-helper.editorTableView`，提供：
- 按类型分组显示物编数据
- 右键菜单：新建、删除、重命名、复制 Key/名称

### Excel 导表（editorTable/excel/）

见 [06-功能流程详解](./06-功能流程详解.md) 的 Excel 导表章节。

## 4. metaBuilder 模块（src/metaBuilder/）

根据用户在物编中定义的枚举数据，自动生成 Lua 代码和 TypeScript 类型声明，使游戏脚本有更好的 IDE 支持。

### 生成流程

```
env 发生变化（地图切换/数据修改）
    │
    └─ 100ms 防抖后，updateAll() 被调用
              │
              ├─ UnitAttrs.update()
              │     读取 customDefine/unitAttrs 数据
              │     生成 meta/unitAttrs.lua：
              │     y3.game_api.unit_attr.xxx = { key = "xxx", name = "显示名称" }
              │
              ├─ PlayerAttrs.update() → meta/playerAttrs.lua
              ├─ CustomEvents.update() → meta/customEvents.lua
              ├─ UI.update() → meta/ui.lua
              ├─ UIAnim.update() → meta/uiAnim.lua
              ├─ JumpWord.update() → meta/jumpword.lua
              ├─ Font.update() → meta/font.lua
              │
              └─ InitBuilder.update()
                    生成 meta/init.lua：
                    include 'y3-helper.meta.unitAttrs'
                    include 'y3-helper.meta.playerAttrs'
                    ...
```

### TS 类型声明生成（tsMeta.ts）

同时生成 `plugin/map-declare.d.ts`，供用户编写插件 JS 时有类型提示。

## 5. plugin 模块（src/plugin/）

允许用户在 `script/y3-helper/plugin/` 目录下放置 `.js` 文件作为插件脚本，在地图保存时或手动触发时自动运行。

### 关键概念

**插件文件** — 标准 JS 文件，通过 `export function` 导出函数：
```javascript
// script/y3-helper/plugin/myPlugin.js
export function onSave() {
    // 地图保存时自动调用
    const y3 = require('y3-helper');
    // 可访问 y3 API
}
export function myCustomFunc() {
    // 用户手动触发
}
```

**执行机制** — 使用 Node.js `vm` 模块在隔离沙箱中运行：
```typescript
vm.runInNewContext(fixedCode, sandbox, { filename: ... })
```

**沙箱可用模块**：
- `y3-helper`（即 y3 模块）
- `vscode`
- `os`, `path`, `fs`, `util`, `child_process`
- 以及相对路径 `require('./xxx')`

### 自动触发时机

- **地图保存**：监听 `{mapDir}/*.gmp` 文件变化，延迟 1 秒后调用所有插件的 `onSave()` 函数
- **启动游戏**：`launchGame` 时调用所有插件的 `onGame()` 函数

## 6. console 模块（src/console/）

游戏运行时与 VSCode 之间的双向通信通道，基于 TCP 协议。

### 通信机制

```
VSCode 侧（ConsoleServer）          游戏侧（Lua）
    TCP Server 监听端口                TCP Client 连接
    port 写入 helper_port.lua  ────→  读取 helper_port.lua 获取端口
                                      连接到 VSCode
    接收 JSON-RPC 请求        ←────   发送 print/command 等请求
    返回 JSON-RPC 响应        ────→   接收响应
```

### 支持的 RPC 方法

| 方法名 | 触发方 | 功能 |
|--------|--------|------|
| `print` | 游戏 | 在 VSCode 终端输出彩色日志 |
| `createTreeView` | 游戏 | 在 VSCode 创建自定义树形视图 |
| `removeTreeView` | 游戏 | 移除自定义树形视图 |
| `refreshTreeNode` | 游戏 | 更新树节点数据 |
| `command` | 游戏 | 执行任意 VSCode 命令 |
| `prepareForRestart` | 游戏 | 准备游戏重启时的调试器重连 |
| `showInputBox` | 游戏 | 弹出 VSCode 输入框 |
| `updatePlayer` | 游戏 | 更新玩家名称（多开模式） |
| `createTracy` | 游戏 | 启动 Tracy 性能分析工具 |
