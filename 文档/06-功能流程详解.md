# 06 - 功能流程详解

## 1. 启动游戏流程

**触发方式**：点击状态栏 `✨启动` 按钮，或按快捷键 `Shift+F5`，或执行命令 `y3-helper.launchGame`

```
用户触发启动游戏
    │
    ├─ 读取 config 配置：
    │     tracy（性能分析）、attachWhenLaunch（启动后附加调试）
    │     multiMode（多开模式）、multiPlayers（多开玩家列表）
    │
    ├─ env.editorReady(true) — 确保编辑器路径已找到（必要时弹对话框）
    ├─ env.mapReady(true) — 确保地图路径已找到（必要时弹对话框）
    │
    ├─ y3.version.askUpdate() — 检查 y3-lualib 是否需要更新
    │     若需要 → 弹出提示，用户确认后返回（不启动游戏）
    │
    ├─ runAllPlugins(map, 'onGame') — 执行所有插件的 onGame 函数
    │     若插件报错 → 弹出确认对话框，用户可选择"仍要启动"
    │
    ├─ updateHelperPortFile() — 将当前 TCP 端口写入 helper_port.lua
    │
    ├─ 构建启动参数：
    │     普通模式：
    │         type@editor_game
    │         subtype@editor_game
    │         editor_map_path@{projectUri}
    │         level_id@{map.id}
    │         release@true
    │         lua_dummy@sp ce
    │         lua_tracy@true（若启用 Tracy）
    │         lua_wait_debugger@true（若启用调试附加）
    │
    │     多开模式：
    │         type@editor_game
    │         subtype@editor_multi_game
    │         role_ids@{player1}#{player2}
    │         level_id@{map.id}
    │         lua_multi_mode@true
    │         lua_multi_wait_debugger@true
    │         lua_multi_debug_players@{debugPlayer1}#{debugPlayer2}
    │
    ├─ runShell(Game_x64h.exe, [
    │     '--dx11',
    │     '--start=Python',
    │     '--python-args={args joined by ,}',
    │     '--plugin-config=Plugins-PyQt',
    │     '--console', '--luaconsole'  (仅普通模式)
    │   ])
    │
    └─ 若 attachWhenLaunch → debug.attach()（附加 Lua 调试器）
```

## 2. 调试附加流程

**触发方式**：点击状态栏 `💡附加` 按钮，或执行命令 `y3-helper.attach`

### 单人模式
```
debug.attach()
    │
    ├─ 停止所有现有调试会话
    └─ attachForOnePlayer()
              ├─ 端口：12399
              └─ vscode.debug.startDebugging({
                      type: 'y3lua',
                      request: 'attach',
                      address: '127.0.0.1:12399'
                  })
```

### 多开模式
```
debug.attach()
    │
    ├─ 停止所有现有调试会话
    └─ 对每个 debugPlayers 中的玩家执行：
              attachForOnePlayer(playerId)
                  ├─ 端口：12399 - playerId
                  └─ vscode.debug.startDebugging({
                          name: '💡附加【playerId】',
                          address: '127.0.0.1:{12399 - playerId}'
                      })
```

### 自动重连机制

```
startWaitDebuggerHelper()（每秒运行一次）
    │
    └─ checkNeedAttach()
              ├─ 遍历所有地图的 .log/wait_debugger 文件
              ├─ 若文件存在 → 删除文件，设 needAttach = true
              └─ 若当前无调试会话且 needAttach → attach()

游戏重启时（游戏侧发送 prepareForRestart RPC）
    │
    └─ prepareForRestart(needDebugger, playerId)
              ├─ 若当前有调试会话 → prepareReconnect（10秒内自动重连）
              └─ 若无调试会话 → 等 2 秒后 attachForOnePlayer()
```

## 3. 初始化项目流程

**触发方式**：执行命令 `y3-helper.initProject`（菜单"初始化项目"）

```
用户触发初始化项目
    │
    ├─ env.mapReady(true) — 确保已找到地图路径（必要时弹对话框）
    │
    ├─ 检查 y3Uri 是否已有 .git 目录
    │     若已初始化 → 报错返回
    │
    ├─ 检查 y3Uri 是否已存在
    │     若存在 → 尝试移到回收站
    │     若删除失败 → 报错返回
    │
    ├─ 选择仓库来源（中文环境）：
    │     "Github（可能需要代理）"：
    │         git clone https://github.com/y3-editor/y3-lualib.git {y3Uri}
    │     "Gitee（国内镜像）"：
    │         git clone https://gitee.com/tsukiko/y3-lualib.git {y3Uri}
    │     英文环境：
    │         git clone https://github.com/CliCli-Editor/lualib.git {y3Uri}
    │
    ├─ 验证 clone 成功（检查 README.md 是否存在）
    │
    ├─ 若编辑器版本为 1.0：
    │     git checkout -b 1.0 origin/1.0
    │
    ├─ 初始化配置：
    │     创建 {scriptUri}/.log/ 目录
    │     从 y3/演示/项目配置/ 复制配置文件到 scriptUri
    │
    ├─ 保存 NewProjectPath 到 globalState
    └─ vscode.openFolder(scriptUri) — 重新打开项目

（VSCode 重新加载后）
    ├─ checkNewProject() 检测到 NewProjectPath 匹配当前工作区
    ├─ 打开 main.lua
    └─ 弹出"欢迎使用Y3编辑器！"
```

## 4. Excel 导表流程

用户将 Excel 文件放到 `script/y3-helper/excel/` 目录，插件自动读取并写入物编数据。

### 规则文件（excel_rule/）

每个 Excel 文件对应一个规则文件，描述如何从 Excel 读取并写入物编：

```javascript
// excel_rule/unit.js（示例）
export function onSave() {
    const y3 = require('y3-helper');
    // 使用 y3 API 读取 excel 并写入物编
}
```

### Sheet 类提供的 API

```typescript
class Sheet {
    // 获取单元格值（如 cells['A1']）
    get cells(): Record<string, string>
    
    // 以某格为锚点，创建 key-value 表格
    // B2 行为 header，往下为数据行，往右为字段
    makeTable(offset?: string, skip?: number): Table
    
    // 创建多值表格（一个 key 对应多行数据）
    makeMultiTable(offset?: string, skip?: number): MultiTable
}
```

### 导表调用流程

```
地图保存（*.gmp 变化）或手动触发
    │
    └─ 执行插件的 onSave() 函数
              │
              ├─ y3.excel.load('myExcel.xlsx')
              │     → 读取 excelUri/myExcel.xlsx
              │     → 返回 ExcelFile 对象
              │
              ├─ excelFile.getSheet('Sheet1')
              │     → 返回 Sheet 对象
              │
              ├─ sheet.makeTable('B2', 1)
              │     → 返回 { key: { field1: val1, field2: val2, ... }, ... }
              │
              └─ 遍历表格，写入物编：
                    y3.editorTable.getTable('单位')
                        .get(key).set('field', value).save()
```

## 5. 插件系统流程

### 插件加载流程

```
PluginManager 初始化（每个 Map 各有一个）
    │
    ├─ loadPlugins()
    │     遍历 {pluginUri}/*.js
    │     为每个文件创建 Plugin 对象（惰性读取代码）
    │
    ├─ createPluginFileWatcher()
    │     监听 {pluginUri}/**/*.js 的创建/删除/修改
    │     实时同步 plugins 字典
    │
    └─ createMapSaveWatcher()
          监听 {mapDir}/*.gmp 文件变化
          延迟 1 秒后触发 runAll('onSave')
```

### 插件执行流程

```
plugin.run(funcName, sandbox)
    │
    ├─ parse() — 读取并预处理代码
    │     原始 JS：export function onSave() { ... }
    │     处理后：function onSave() { ... }
    │              module.exports = { onSave }
    │
    ├─ vm.runInNewContext(fixedCode, sandbox)
    │     在隔离沙箱中执行代码
    │     sandbox 提供 require / exports / module
    │
    └─ exports[funcName]() — 调用目标函数
```

## 6. Lua 文档生成流程

**触发方式**：执行命令 `y3-helper.makeLuaDoc`

```
LuaDocMaker.make()
    │
    ├─ 找到 sumneko.lua 扩展的 lua-language-server 可执行文件
    │
    ├─ runShell(lua-language-server, [
    │       '--doc={y3Uri}',          // 扫描 y3/ 目录
    │       '--logpath={logUri}',     // 输出日志到全局存储目录
    │       '--locale=zh-cn'
    │   ])
    │
    ├─ 读取 doc.json 解析所有类和字段信息
    │
    ├─ clipDoc() — 过滤：
    │     排除以 _ 开头的私有类
    │     排除 GameAPI / GlobalAPI / py. 等内部 API
    │     排除名称以大写字母或数字结尾的类
    │     只保留在 y3/ 目录内定义的符号
    │
    ├─ splitDoc() — 按类名分组（y3.Const → Const 组）
    │
    ├─ makeMenu() — 生成 y3/doc/API.md（目录页）
    │
    └─ convertDocToMarkdown() — 生成各类的 Markdown 文件
          写入 y3/doc/API/{ClassName}.md
```

## 7. ECA 编译流程

**触发方式**：执行命令 `y3-helper.compileECA`

ECA（触发器条件事件动作）编译将编辑器中的触发器逻辑转换为 Lua 代码。

```
用户触发 ECA 编译
    │
    ├─ env.mapReady() — 确保地图已加载
    │
    ├─ fillStatic(formatter) — 加载静态配置（内置函数定义等）
    ├─ fillMapDefined(formatter) — 加载地图定义（自定义事件/属性等）
    ├─ fillFromLocalExcel(formatter) — 从 Excel 读取额外配置
    │
    └─ Process.fullCompile()
              │
              ├─ 遍历触发器 JSON 文件（{triggerMapUri}/editor_table/trigger/）
              ├─ 对每个触发器调用 compiler.compile(trigger)
              │     解析条件/事件/动作节点
              │     通过 formatter 格式化为 Lua 代码
              └─ 写入输出文件
```

## 8. 全局脚本功能流程

全局脚本允许多个地图共享同一份 Lua 代码库。

```
用户启用全局脚本（globalScript.enable()）
    │
    ├─ 检查 globalScriptUri 是否存在
    ├─ 将 entryMap 的 y3/ 目录复制到 globalScriptUri/y3/
    ├─ 遍历所有地图，修改各自的 .luarc.json：
    │     添加 runtime.path: ["../../../global_script/?.lua", ...]
    │     添加 workspace.library: ["../../../global_script"]
    ├─ 生成 globalScriptUri/.luarc.json
    └─ 删除各地图自己的 y3/ 目录（改为共享的）

（env 变化时）
    └─ openGlobalScript()
              若全局脚本已启用 → 将 globalScriptUri 加入工作区
              使开发者能在同一 VSCode 窗口编辑地图脚本和全局脚本
```
