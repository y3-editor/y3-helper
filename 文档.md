## 1. 项目简介

**Y3 开发助手**（`y3-helper`）是一个 **VSCode 插件**，用于辅助 Y3 编辑器（游戏编辑器）的开发工作。它提供以下核心功能：

- 初始化 Y3 Lua 开发环境（自动从 GitHub/Gitee 克隆 lualib）
- 物编数据（editor_table）浏览、编辑、重命名、导表
- 启动游戏、启动编辑器、Lua 调试附加
- 插件管理（初始化、更新、运行插件）
- ECA（触发器）编译为 Lua
- Lua 语言增强支持（代码跳转、提示、补全）
- 自定义视图（单位属性、玩家属性、自定义事件、UI 动画等）
- Excel 导表支持

插件发布在 [VSCode 扩展市场](https://marketplace.visualstudio.com/items?itemName=sumneko.y3-helper)，并在 [GitHub](https://github.com/y3-editor/y3-helper) 上开源。

## 3. 项目整体架构

```
y3-helper-inner/
├── src/                    # TypeScript 源码（核心）
├── 3rd/                    # 第三方库（Lua 调试器等）
├── template/               # 模板文件（生成给用户使用）
├── l10n/                   # 国际化翻译文件（英文 bundle）
├── image/                  # 图标资源
├── dist/                   # webpack 打包产物（编译输出，不提交 git）
├── out/                    # tsc 编译输出（仅用于类型生成，不提交 git）
├── tmp/                    # 类型声明临时输出目录（不提交 git）
├── .github/workflows/      # GitHub Actions CI/CD 配置
├── package.json            # 插件清单 + npm 依赖
├── tsconfig.json           # TypeScript 编译配置
├── webpack.config.js       # webpack 打包配置
└── .vscodeignore           # 打包 vsix 时的排除规则
```

### 技术栈

| 技术 | 用途 |
|------|------|
| **TypeScript** | 全部源码语言 |
| **webpack 5** | 将源码打包为单一 `dist/extension.js` |
| **ts-loader** | webpack 中负责编译 TypeScript |
| **VSCode Extension API** | 插件运行时 API |
| **@vscode/l10n** | 国际化（中英文支持） |
| **exceljs** | Excel 读取（导表功能） |
| **winreg** | 读取 Windows 注册表（自动寻找编辑器路径） |

## 7. 编译方式与流程

### npm 脚本一览

| 命令 | 说明 |
|------|------|
| `npm run compile` | 开发模式编译（不压缩，含 source map） |
| `npm run watch` | 监听模式（文件变化时自动重新编译） |
| `npm run vscode:prepublish` | 生产模式编译（用于打包 vsix，压缩代码） |
| `npm run lint` | ESLint 代码检查 |

### 编译流程详解

```
TypeScript 源码 (src/)
        ↓
   ts-loader（将 .ts 编译为 JS）
        ↓
   webpack 5（打包所有模块为单一文件）
        ↓
   dist/extension.js    ← 插件主文件
   dist/1.extension.js  ← 异步加载的 chunk（动态 import 部分）
```

**关键配置说明：**

- **入口（entry）**：`src/extension.ts`
- **输出（output）**：`dist/extension.js`，使用 `commonjs2` 格式（Node.js 模块）
- **目标（target）**：`node`（插件运行在 Node.js 环境，不是浏览器）
- **externals**：`vscode` 模块不打包（由 VSCode 宿主提供）
- **路径别名**：`y3-helper` → `src/y3-helper.ts`（通过 `tsconfig-paths-webpack-plugin`）

### `tsc` 单独编译的用途

`tsconfig.json` 中的 `tsc` 编译（`tsc -p ./`）主要用于：
- 生成 `.d.ts` 类型声明文件到 `tmp/types/`（供 `dts-bundle-generator` 使用）
- 生成测试相关的 JS 到 `out/`

> **注意**：插件的实际运行代码是由 **webpack** 编译生成的 `dist/extension.js`，不是 `tsc` 的产物。

## 8. 打包为 .vsix 安装包（内网版）

内网版本不走 GitHub Actions，需要**本地手动打包**。

### 前置准备

```bash
# 全局安装 vsce（VSCode Extension 打包工具）
npm install -g @vscode/vsce

# 全局安装 TypeScript（publish 流程需要）
npm install -g typescript
```

### 打包步骤

```bash
# 第一步：进入项目目录
cd d:\github\y3-helper-inner

# 第二步：安装依赖（如未安装）
npm install

# 第三步：生产模式编译（自动在 vsce package 前触发，或手动执行）
npm run vscode:prepublish

# 第四步：打包为 .vsix
vsce package

# 打包成功后会生成：y3-helper-{版本号}.vsix
# 例如：y3-helper-1.21.6.vsix
```

> `vsce package` 会自动调用 `vscode:prepublish` 脚本（即 `webpack --mode production`），所以第三步也可以省略。

### 安装 .vsix 到 VSCode

```bash
# 方法一：命令行
code --install-extension y3-helper-1.21.6.vsix

# 方法二：VSCode 界面
# 扩展面板 → 右上角 "..." → "从 VSIX 安装..."
```

### 打包时的文件排除规则

`.vscodeignore` 定义了哪些文件**不会**被打进 vsix：

```
.vscode/          ← 编辑器配置
src/              ← TypeScript 源码（打包后不需要）
!src/**/*.json    ← 但保留 src 中的 JSON 文件（如 helper_meta 的字段描述）
node_modules/     ← 依赖（已打包进 dist/extension.js）
out/              ← tsc 编译产物（不需要）
tmp/              ← 类型声明临时目录
.github/          ← CI 配置
```

最终 vsix 中包含的关键内容：
- `dist/extension.js` — 编译打包后的插件主体
- `3rd/` — 第三方 Lua 调试器
- `template/` — 模板文件
- `l10n/` — 国际化文件
- `image/` — 图标
- `package.json` — 插件清单

## 9. GitHub 上的自动化发布原理（公开版）

GitHub 上配置了两条 CI/CD 流水线（`.github/workflows/`）：

### 流水线一：`build.yml` — 每次推送 main 分支时触发

**触发条件**：向 `main` 分支推送代码（不含 tag）

**流程**：
1. 检出代码
2. 安装 Node.js >= 20
3. `npm install` + `tsc -p ./`（编译）
4. `vsce package` 打包为 `.vsix`
5. 将 vsix 上传为 **GitHub Actions Artifact**（供下载测试，不发布）

### 流水线二：`publish.yml` — 推送 tag 时触发（正式发布）

**触发条件**：推送任意 tag（如 `v1.21.6`）

**完整发布流程**：

```
推送 tag
    ↓
GitHub Actions 启动（Windows 环境）
    ↓
1. 检出 main 分支代码
2. npm install + tsc 编译
3. vsce package 打包为 y3-helper-{tag}.vsix
4. git push（如有 package-lock.json 变动则提交）
    ↓
5. 上传 vsix 为 Artifact
6. 创建 GitHub Release（自动生成 Release Notes）
   + 将 vsix 附加到 Release
    ↓
7. vsce publish 发布到 VSCode 扩展市场
```

### 发布新版本的操作步骤（GitHub 公开版）

```bash
# 1. 修改 package.json 中的 version 字段，如改为 "1.21.7"

# 2. 提交并推送到 GitHub
git add package.json
git commit -m "1.21.7"
git push github main

# 3. 打 tag 并推送（此步骤触发自动发布）
git tag 1.21.7
git push github 1.21.7

# GitHub Actions 会自动完成后续的打包、发布到市场等操作
```

### 所需 Secrets（GitHub 仓库设置中配置）

| Secret 名称 | 用途 |
|-------------|------|
| `Y3_COMMIT_TOKEN` | GitHub Personal Access Token（用于创建 Release、推送代码） |
| `VSCODE_TOKEN` | VSCode 扩展市场 Personal Access Token（用于发布插件） |

## 10. 将内网改动同步到 GitHub

内网日常开发完成后，需要手动将代码同步到 GitHub 公开仓库。

### 首次配置（只需一次）

```bash
# 添加 GitHub 为第二个远端
git remote add github https://github.com/y3-editor/y3-helper.git
```

### 日常同步

```bash
# 正常开发：提交到内网
git add .
git commit -m "你的改动描述"
git push origin main   # 推送到内网

# 同步到 GitHub
git push github main
```

### 发布公开版本（tag 触发自动发布）

```bash
# 打 tag 并同步到 GitHub（会自动触发 publish.yml）
git tag 1.21.7
git push github 1.21.7
```

> ⚠️ **注意**：不要将 tag 推送到内网 `origin`，tag 只用于触发 GitHub 的自动发布流程。

## 11. 常见问题

### Q：`npm audit` 报告漏洞怎么处理？

不要直接用 `npm audit fix --force`，因为它会自动决策版本而可能引入兼容性问题。正确做法：

1. 查看 `npm audit` 输出，确认漏洞来自哪个包
2. 手动在 `package.json` 中升级对应的 `devDependencies` 版本
3. 若是子依赖的漏洞（自己无法直接控制版本），在 `package.json` 中添加 `overrides` 字段强制覆盖：
   ```json
   "overrides": {
       "serialize-javascript": ">=7.0.3",
       "diff": ">=8.0.3"
   }
   ```
4. 执行 `npm install` 后再次验证 `npm audit`

### Q：编译后出现 TypeScript 类型错误怎么办？

`@types/node` 升级后对 `Buffer`、`Uint8Array` 等类型的泛型更严格。常见解法：

- 对 `Uint8Array` 赋值问题：使用 `as Uint8Array<ArrayBuffer>` 断言
- 对 `Buffer` 类型不兼容问题：将 getter 返回值声明为 `any` 绕过

### Q：如何更新插件版本号？

只需修改 `package.json` 中的 `version` 字段，无需修改其他地方。

### Q：内网打包的 vsix 如何分发给团队？

将打包好的 `.vsix` 文件通过内部文件共享（如企业网盘、内部文档系统等）分发给团队成员，成员在 VSCode 中通过"从 VSIX 安装"进行安装。

### Q：`dist/` 目录需要提交到 git 吗？

不需要。`dist/` 已在 `.gitignore` 中排除，每次需要使用时本地执行 `npm run compile` 或 `npm run vscode:prepublish` 即可重新生成。打包 vsix 时 `vsce` 会自动触发 `vscode:prepublish` 脚本生成 `dist/`。
